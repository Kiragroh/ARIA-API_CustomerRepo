Resource Profile: Patient - Search Parameters

Following are the Search Parameters for the Patient Profile:

# Name Type Description
1 _has string Supports reverse chaining - that is, selecting resources based on the properties of resources that refer to them. Supported target resources are Appointment, CareTeam, Flag. For Appointment eg:'_has:Appointment:patient:date=2015-10-08', | For CareTeam Department participant eg:'_has:CareTeam:patient:participant=Organization/Organization-Dept-1' | For CareTeam Practitioner participant eg:'_has:CareTeam:patient:participant=Practitioner/Practitioner-1004', | For Flag eg:'_has:Flag:patient:code='1''
2 _id token The ID of the resource
3 active token Search Active Patients only. Set value to: true | false
4 address string A server defined search that may match any of the string fields in the Address, including line, city, district, state, country, postalCode, and/or text
5 address-city string A city specified in an address
6 address-country string A country specified in an address
7 address-postalcode string A postalCode specified in an address
8 address-state string A state specified in an address
9 birthdate date The patient's date of birth
10 email token A value in an email contact
11 family string A portion of the family name of the patient
12 gender token Gender of the patient
13 given string A portion of the given name of the patient
14 identifier token Supports 'AND' search with listOr param for the different identifiers. e.g. .../fhir/Patient?identifier=http://varian.com/fhir/identifier/Patient/ARIAID1|ID1,identifier=http://varian.com/fhir/identifier/Patient/SSN|123456789&identifier=http://varian.com/fhir/identifier/Patient/ARIAID2|ID2
15 mothersMaidenName string Mother's Maiden Name
16 name-or-identifier string Free text search in PatientId1 | PatientId2 | LastName | FirstName
17 phone token A value in a phone contact
18 primary-practitioner reference Primary Practitioner of Patient
19 telecom token The value in any kind of telecom details of the patient
20 testPatient token Search Test Patients only. Set value to: true | false
