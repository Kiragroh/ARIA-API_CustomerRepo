Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide.

A reference to a document

This resource identifies the healthcare documentation created for a Patient in ARIA
ActivityDefinition

This resource allows for the definition of some activity to be performed, it may be used to specify actions to be taken as part of a workflow, appointment or tasks.
AllergyIntolerance

AllergyIntolerance represents Patient’s Allergies in ARIA.

All recorded allergies or NO KNOWN ALLERGY status will be reflected using this.
Appointment

A booking of a healthcare event among patient(s), practitioner(s), related person(s) and/or device(s) for a specific date/time. This may result in one or more Encounter(s).
Audit Event

A record of an event made for purposes of maintaining a security log. Typical uses include detection of intrusion attempts and monitoring for inappropriate usage.
BodyStructure

This resource record details about an anatomical structure. This resource may be used when a coded concept does not provide the necessary detail needed for the use case.
CarePlan

Care Plan profile is derived from FHIR Domain Resource. It represents care plan in ARIA .
CareTeam

This Resource includes all the people and organizations who plan to participate in the coordination and delivery of care for a patient.

Careteam represents department and practitioner (doctor/staff) assigned to the Patient in ARIA.

ARIA does not configure predefined teams, so single Careteam per Patient will represent associations with Department and practitioner.
ChargeItem

This Resource represents charges captured for an activity.
Condition-Diagnosis (CancerCondition)

This resource is used to record detailed information about a diagnosis that has risen to a level of concern. The condition could be a point in time diagnosis in context of an encounter, it could be an item on the practitioner’s Problem List, or it could be a concern that doesn’t exist on the practitioner’s Problem List.
Device

This resource identifies an instance or a type of a manufactured item that is used in the provision of healthcare without being substantially changed through that activity. The device may be a medical or non-medical device. Medical devices include durable (reusable) medical equipment, implantable devices, as well as disposable equipment used for diagnostic, treatment, and research for healthcare and public health. Non-medical devices may include items such as a machine, cellphone, computer, application, etc.
Group

Group represents logical grouping of Resources like Advance Practitioners, Nurses,Oncologists, System administrators etc…
Group comprises Doctor/Staff (Practitioner), Machine (Device) and Venue (Location) with in a department in ARIA.
HealthcareService

HealthcareService represents departments in ARIA as in ARIA department and services are not two separate concepts. Although Department are already being represented by Organization resource, since many other FHIR clinical resources works on HealthcareService concept, we are mapping department in Healthcare service as well.
Location

Location includes both incidental locations (a place which is used for healthcare ) and dedicated, formally appointed locations. It can be of Auxiliary type or Venue type. Auxiliary locations include imaging machines and treatment machines. Venues are locations within a department like treatment room, waiting room, consultation room, etc. Some locations can be schedulable.
Observation-StageCategory

Observations are a central element in healthcare, used to support diagnosis, monitor progress, determine baselines and patterns and even capture demographic characteristics. Most observations are simple name/value pair assertions with some metadata, but some observations group other observations together logically, or even are multi-component observations.

This profile is used to represent the Stage Category in Condition Diagnosis resource.
Observation-StageGroup

Observations are a central element in healthcare, used to support diagnosis, monitor progress, determine baselines and patterns and even capture demographic characteristics. Most observations are simple name/value pair assertions with some metadata, but some observations group other observations together logically, or even are multi-component observations.

This profile is used to represent the Stage Group in Condition Diagnosis resource.
Observation-Vital-Signs

Vital Signs profile is derived from FHIR Observation.

It represents Patient’s Vital Signs Observations recorded in ARIA .
Observation-Vital-Signs-Component

Vital Signs component profile is derived from FHIR Observation.

It represents Patient’s individual Vital Signs result Observations recorded in ARIA .
Organization

A formally or informally recognized grouping of organizations formed for the purpose of achieving some form of collective action. Includes institutions, departments etc

Organization represents Hospital and Departments configured in ARIA. A Hospital can have multiple departments.

Hospital is represented with category ‘prov (Provider)’ and Department is represented with category ‘dept (Department)’.

Hospital and Department relationship is represented on Department.
Patient

This profile represents Patient Demographics, Contacts and other administrative information. Profile Contains: Demographics, Contacts, Region specific details like Race, Ethnicity and Language (if applicable), User Defined Labels
Practitioner

Practitioner covers all individuals who are engaged in the healthcare process and healthcare-related services as part of their formal responsibilities and this Resource is used for attribution of activities and responsibilities to these individuals. Practitioners include (but are not limited to):

physicians, dentists, pharmacists, physician assistants, nurses, scribes, midwives, dietitians, therapists, optometrists, paramedics, medical technicians, laboratory scientists, prosthetic technicians, radiographers, social workers, professional homecare providers, official volunteers, receptionists handling patient registration

Practitioner represents Doctor and Staff configured in ARIA.

Doctor/Staff demographics, role and association with department is being represented.
Radiotherapy Course Prescription

The elements of a treatment course that can be defined during prescription. This represents the treatment course in ARIA, not an actual prescription and is the parent for all presciptions in the course. It contains the data elements of a course that are not specific to planning or delivery, for example, treatment intent, relation to the diagnosis, or course status. Those elements are exposed as CodeX RT Radiotherapy Course Prescription although ARIA does not have an actual prescription for the complete course. See the Phase Prescription profile for actual prescriptions in ARIA. With the Radiotherapy Course Prescription, elements of the course can be found as soon as a course is defined in ARIA, even if no treatment plans or delivered treatments are available yet.
Radiotherapy Course Summary

A summary of the treatment progress in a radiotherapy course of treatment. When additional treatments are delivered in the same course, this resource is updated. No new resource is created.
Radiotherapy Phase Prescription

A prescription for a radiotherapy treatment phase. In ARIA, a prescription can cover a single plan or complete phase implemented with one or multiple plans. During prescription it may not even be defined whether multiple plans will be used. Therefore, all ARIA prescriptions are exposed as Phase Prescriptions.
Radiotherapy Planned Course

A summary of all plans in a radiotherapy course of treatment. Only approved plans with status other than Unapproved, Rejected, Reviewed are considered.
Radiotherapy Planned Phase

A summary of all plans in a radiotherapy treatment phase. In ARIA, all plans linked to the same prescription are considered as part of the same phase. One Radiotherapy Planned Phase is exposed for each ARIA prescription that has at least one linked plan in an approved status (other than Unapproved, Rejected, Reviewed).
Radiotherapy Treated Phase

A summary of the treatment progress in a radiotherapy treatment phase. In ARIA, all plans linked to the same prescription are considered as part of the same phase.
Radiotherapy Treated Plan

A summary of the treatment progress in one radiotherapy treatment plan.
Radiotherapy Treatment Plan

A radiotherapy treatment plan.
Radiotherapy Volume

A volume of the body used in radiotherapy planning or treatment delivery. In ARIA, dose is specified and tracked against Reference Points. These Reference Points can represent actual points or three-dimensional volumes. ARIA calls all of those ‘Reference Point’, even if not a point. Each Reference Point in ARIA is exposed as a Radiotherapy Volume.
Task

Task resource represents tasks configured in ARIA.

A task resource describes an activity that can be performed and tracks the state of completion of that activity. It is a representation that an activity should be or has been initiated, and eventually, represents the successful or unsuccessful completion of that activity.
ValueSet

  • ValueSet specifies a set of codes drawn from one or more code systems, intended for use in a particular context. Value sets link between CodeSystem definitions and their use in coded elements.

  • ^publisher = “Varian ARIA API Profile Team”

  • Value sets have 2 aspects:

    • .compose: A definition of which codes are intended to be in the value set (“intension”) -/li-
    • .expansion: The list of codes that are actually in the value set under a given set of conditions (“extension”) -/li-

  • Please refer the How to -> Valueset & Codesystem List Page for list of valueset and codesystems.

Terminology: Value Sets

These define sets of codes used by systems conforming to this implementation guide.

AllergyIntolerance ARIA Category

This value set defines different allergy types configured in Aria. List of configured allergy categories are available under valueset ‘aria-allergy-category’. All values can be accessed using $expand operation i.e. /fhir/ValueSet/$expand?url=http://varian.com/fhir/ValueSet/allergy-intolerance-category.
CarePlan Template

The configured careplan template in Aria. List of configured templates are accessible using valueset ‘aria-careplan-template’. All values can be accessed using $expand operation i.e. /fhir/ValueSet/$expand?url=http://varian.com/fhir/ValueSet/careplan-template

Diagnosis Code
List of available codes are accessible using valueset ‘aria-diagnosiscode’. All values can be accessed using ValueSet $expand operation i.e. /fhir/ValueSet/$expand?url=http://varian.com/fhir/ValueSet/condition-code. Codes can be filtered on schemas using context filter i.e. /fhir/ValueSet/$expand?url=http://varian.com/fhir/ValueSet/condition-code&context=http://hl7.org/fhir/sid/icd-9-cm icd-9-cm
